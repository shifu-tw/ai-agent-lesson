★ 心智模型

你要建的不是工具，是夥伴

工具

每次重新解釋背景。換話題從頭來。關掉全忘。你遷就它。

vs
夥伴

知道你的公司、思考方式、上次的決定。你只說新的事。它遷就你。

今天建的不是設定好就不動的系統。是一個活的、會跟著你進化的夥伴。

★ 底層邏輯

Context Window — AI 的工作記憶

每次對話有一個「工作記憶」上限。

就算有 100 萬 token，塞越滿越笨。關鍵資訊被雜訊淹沒。

Context 污染

塞不相關的東西 = 稀釋 AI 注意力。像開會時旁邊有人一直講無關的事。

⚠
塞滿的後果

回答變泛、忘記前面說的、開始幻覺、語氣變奇怪、重複自己

所有設計都在解決同一個問題：讓 AI 有限的工作記憶裡，永遠放著最重要的東西。

不做的後果

如果不建...會怎樣

MEMORY.md上週討論三小時的策略，今天全忘了。永遠原地打轉。

vault/你的文件、會議紀錄都用不上。只靠訓練資料回答。

🚀 作業

這一週：用爆它

唯一規則：本來在做的事，全部先問它。

  • 收到信 → 先讓它讀
  • 開完會 → 跟它說剛聊什麼
  • 要寫東西 → 讓它先起草
  • 做決策前 → 把脈絡丟給它
  • 讀到文章 → 丟進 vault/
  • 想到什麼 → 直接跟它說
  • 做完重要事 → /commit 備份

記得：不同任務開不同對話。長對話品質會衰退。

☑ 驗收

一週後

跟 Agent 說「跑 review」。下次上課看結果。

  • MEMORY.md 至少 15 條記憶
  • vault/ 至少 5 份文件
  • 起草 3+ 內容、整理 3+ 會議
  • gws 至少用 10 次
  • 問「你了解我什麼？」準確 10+ 點
  • 它寫的 60%+ 不用大改
  • 糾正 3+ 次且沒再犯
  • 能說出「它最懂我的一件事」
📚 深入

Skill 的本質

Skill = 一份 SKILL.md。寫給 AI 讀的 SOP。

你教過一次怎麼做某件事，不應該每次重新教。Skill 就是那個「教一次就好」。

📄
觸發條件

什麼時候啟動

⚙
執行步驟

做什麼、怎麼做

★
品質標準

好的產出長什麼樣

Context 策略：Skill 不會一直佔 context。只有被觸發時才載入。50 個 Skill 只載用到的那幾個。

👥 Skill

同事.skill — 蒸餾真實的人

GitHub 上 5000+ 星。證明蒸餾一個人完全可行。
