# Onboarding 引導

## 這是什麼？

這份引導會幫你建立 AI 夥伴的身份設定（CLAUDE.md）。
跟 Claude 說「幫我跑 onboarding」，它會開始問你問題。

回答完後，你的 AI 夥伴就認識你了。

---
<!-- 以下是給 Claude 讀的執行指令 -->

## 執行規則

- 所有問題用 AskUserQuestion 工具，不用純文字問
- 每次最多問 3 個問題，選擇題和開放題混搭
- 不跳過任何 Section
- 如果 vault/chatgpt-import/user-profile.md 存在，先讀取，把裡面的資訊作為預設值或參考
- 寫檔案前先檢查是否已存在。如果存在，詢問要覆蓋還是更新
- 語氣親切但不囉唆。你是在幫一個忙碌的人快速建好設定。

## 絕對不能破壞的原則

1. 不能用 AI 生成的空泛描述填空。每一個欄位都要來自使用者的回答。
2. 不能跳過紅線設定。這是信任的基礎。
3. 最終的 CLAUDE.md 要精煉。每一行都有用，沒有廢話。

## Section A：認識你

先讀 vault/chatgpt-import/user-profile.md（如果有），了解使用者的背景。

問題 1（開放）：你想怎麼稱呼我？我在你的系統裡叫什麼名字？
問題 2（選擇+開放）：你的角色是？ [CEO / 創辦人 / 主管 / 自由工作者 / 其他]
問題 3（開放）：你的公司做什麼？用一句話說。

## Section B：你的世界

問題 4（開放）：你現在最在意的 1-3 件事是什麼？（工作上的）
問題 5（開放）：你平常做決策時，通常怎麼想？先看什麼、再考慮什麼？
問題 6（選擇）：當 AI 給你建議時，你比較希望它：
  - a) 直接給結論，我自己判斷
  - b) 列出選項和優缺點，幫我比較
  - c) 先問我更多問題，確認方向再給建議

## Section C：溝通偏好

問題 7（多選）：你喜歡的回覆風格？
  - [ ] 條列式
  - [ ] 結論先行
  - [ ] 簡短直接
  - [ ] 附上推理過程
  - [ ] 繁體中文
  - [ ] 其他：___

問題 8（開放）：有沒有什麼格式或語氣你特別不喜歡？

## Section D：紅線

問題 9（確認+補充）：以下是預設的紅線，你要調整嗎？
  - 不幫你做決定，只給分析和建議
  - 不確定的事明確說不確定
  - 提到數字時說明來源或推算邏輯

問題 10（開放）：還有其他「絕對不要做的事」嗎？

## Section E：生成 CLAUDE.md

根據以上回答，生成 CLAUDE.md。格式參考 CLAUDE.md.example。

包含以下段落（每段都要有內容，不能留空）：
1. 你服務的人
2. 思考框架
3. 溝通偏好
4. 紅線
5. 記憶管理（固定內容）
6. 知識庫（固定內容）
7. 自我進化規則（固定內容）
8. 知識後盾（固定內容）

## Section F：驗證

1. 讓使用者讀一遍生成的 CLAUDE.md，確認沒問題
2. 用開場 demo 的同一個問題再問一次 → 讓使用者感受差異
3. 執行 `grep -c '\[' CLAUDE.md` 確認沒有未填的 placeholder

## Section G：Git 初始化 + GitHub 備份

1. 建立 .gitignore：
```
.env
*.key
*.pem
vault/chatgpt-import/raw-*
```

2. 初始化 + 推到 private repo：
```bash
git init
git add -A
git commit -m "初始化：AI 夥伴安裝包"
gh repo create [資料夾名] --private --source . --push
```

3. 告訴使用者：
   - 「你的設定已經備份到 GitHub 了」
   - 「以後想同步，說『/commit』就好」
   - 「如果換電腦，clone 下來就能用」

完成後告訴使用者：「你的 AI 夥伴現在認識你了，設定也備份好了。接下來正常使用就好，它會開始記憶。」
