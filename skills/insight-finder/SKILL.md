# Insight Finder

## 觸發條件

使用者說以下任一：
- 「找 insight」
- 「分析一下這個」
- 「有什麼我沒注意到的」
- 「整理一下重點」
- 「這份資料的 insight 是什麼」
- 丟進一份文件、一段文字、或一個會議紀錄

## 輸入

可以是：
- 一段貼進來的文字
- vault/ 裡的一份文件
- 口述的會議內容
- 一個網址（用 CLI 抓內容）

## 執行步驟

### Step 1：整理脈絡

讀完素材後，先整理：
- **主題**：這份素材在講什麼
- **關鍵人物 / 組織**：提到了誰
- **時間軸**：事件的順序
- **立場**：素材的觀點偏向

不要摘要。要結構化整理。

### Step 2：標出關鍵訊號

找出素材中的：
- **重複出現的關鍵詞或概念**（高頻 = 重要）
- **數字和比例**（具體的數據點）
- **轉折點**（「但是」「然而」「不過」後面的內容通常是重點）
- **缺席的資訊**（該提但沒提的東西）

### Step 3：挖隱藏 insight

基於 Step 1-2，提出：
- **表面在說什麼 vs 真正在說什麼**
- **隱含的假設**（這份素材預設了什麼是對的？）
- **矛盾或不一致**（A 段說 X，但 B 段的數據暗示 Y）
- **連結**（這跟使用者最近在關注的事有什麼關係？參考 CLAUDE.md 和 MEMORY.md）

### Step 4：產出

格式：
```
## 脈絡整理
[結構化的脈絡]

## 關鍵訊號
- [訊號 1]
- [訊號 2]
- ...

## 隱藏 Insight
1. [insight 1 — 為什麼這很重要]
2. [insight 2 — 為什麼這很重要]
3. [insight 3 — 為什麼這很重要]

## 值得注意
- [矛盾、風險、或缺席的資訊]

## 跟你的關聯
[結合 CLAUDE.md 的脈絡，這對你的工作/決策意味著什麼]
```

## 品質標準

- insight 不是摘要。「重點是 X」不是 insight。「表面是 X，但隱含 Y」才是。
- 每個 insight 都要說明為什麼重要。
- 最後一段「跟你的關聯」是整個 Skill 最有價值的部分 — 要結合使用者的脈絡。
- 如果素材太短或太淺，沒有足夠的 insight 可挖，誠實說。不要編。
